SonarCloud / SonarQube Connector for Confluence

Integra tus métricas de SonarQube o SonarCloud en tus páginas de Confluence.

Con esta aplicación podrás añadir y consultar tus métricas de SonarQube o SonarCloud sin salir de Confluence y consultar el estado real de calidad de código rápidamente.

PRUÉBALA GRATIS  

¿Por qué utilizar esta app?

A continuación te presentamos las acciones que puedes realizar con SonarQube/SonarCloud Connector for Confluence y los beneficios que vas a obtener.

Visualización de Macros

Con este conector podrás visualizar la información general de 1:N proyectos (tamaño, bugs, vulnerabilidades, code smells, cobertura de código y código duplicado).

Información desglosada

Esta aplicación te permitirá añadir la información de una métrica específica para visualizarla. También podrás consultar las issues de SonarQube o SonarCloud de uno o varios autores.

Datos críticos

Gracias a esta solución podrás visualizar una lista de issues críticas y bloqueantes (bugs, vulnerabilidades y code smells) de un proyecto de tu SonarQube o SonarCloud.

Integra la calidad en tu documentación

Gracias al conector de SonarQube/SonarCloud agilizarás el proceso de consulta del estado real de tu código y te será más fácil tomar decisiones para mejorar el estado de tu código. Todo ello sin salir de tu página de Confluence

La aplicación SonarQube/SonarCloud Connector para Confluence te permitirá tener acceso a los datos más relevantes de la calidad del código en el mismo espacio donde reside toda la documentación en Confluence.


Esta integración, gracias a macros dinámicas, permite traer la información de SonarQube o SonarCloud a Confluence. De esta forma, se agiliza el proceso de consulta previo a la toma de decisiones.

Ahora, los usuarios podrán asegurarse de si la calidad del código es suficientemente buena directamente desde Confluence.

¿Qué métricas podrás visualizar?

Fiabilidad

Incluye las evidencias de tipo bug. Un bug representa algo que está mal en el código. Si el software aún no se ha roto, se romperá, y probablemente en el peor momento. Todos los bugs tienen que solucionarse cuanto antes.

Seguridad

Incluye las evidencias de tipo vulnerabilidad, que representan una potencial puerta trasera para atacantes. Aquí se van a encontrar todas las evidencias que tengan una mínima sospecha de afectar a la seguridad. Es responsabilidad del equipo eliminar los falsos positivos y centrarse en resolver los casos reales.

Mantenibilidad

Incluye las evidencias de tipo code smells, que al final son todas esas malas prácticas que a la larga van a provocar que cada vez sea más difícil hacer cambios en el código. En el peor de los casos, todos estos code smells van a hacer que los desarrolladores se confundan, lo que terminará introduciendo nuevos bugs.

Además podrás acceder fácilmente a:

  • Densidad de código duplicado
  • Líneas de código
  • Deuda técnica
  • Ratio de deuda técnica
  • Cobertura de tests unitarios
  • Densidad de comentarios
