Ya puedes **actualizar, asignar y crear tareas de Jira** sin salir de un canal de Slack. Muchas veces mientras hablamos por Slack con algún/a compañero/a pueden aparecer informaciones que podrían ser **añadidas en Jira**. Para ello, deberíamos ir a la app y crear o actualizar la tarea. ¡Pues ya no! Con la aplicación **Jira Cloud para Slack**, ahora puedes **mencionar a @Jira** en cualquier canal de Slack para crear, actualizar y asignar elementos de trabajo utilizando lenguaje natural. En este artículo te contamos paso a paso cómo puedes hacerlo. 

<h2>Requisitos previos</h2>

Antes de comenzar, es importante que tengas en cuenta que debes cumplir estos requisitos:

- Revisa que [Rovo](https://www.excentia.es/que-es-atlassian-rovo) **está habilitado para tu instancia**.
- Debes tener la app **Jira Cloud para Slack instalada** y habilitada por el administrador de tu sitio.
- **Acceso al sitio de Jira** configurado para tu canal de Slack.

<h2>Guía paso a paso</h2>

- Menciona @Jira **en cualquier canal de Slack**. Si la aplicación aún no se ha añadido al canal, sigue las indicaciones para invitarla. La primera vez que la utilices tendrás que seleccionar tu instancia de Jira y el espacio predeterminado que desees.

- Menciona @Jira y **escribe lo que necesitas en un lenguaje natural** para crear o actualizar elementos de trabajo, con todos sus campos y una tarjeta editable en el hilo.

Si quieres **cambiar la configuración en cualquier momento, escribe /jira agent** en el canal para actualizar tu sitio, tu espacio o añadir una instrucción personalizada.

<h2>Casos de uso</h2>

A continuación, te dejo algunos ejemplos que puedes poner en práctica para utilizar Jira Cloud para Slack:

| Acción que puedes realizar desde Slack            | ¿Se puede hacer? | Ejemplo de comando                                                 |
| ------------------------------------------------- | :--------------: | ------------------------------------------------------------------ |
| Crear una tarea de Jira                           |       ✅ Sí       | @Jira crea una tarea para revisar la documentación del proyecto  |
| Crear un bug                                      |       ✅ Sí       | @Jira crea un bug para el error mencionado anteriormente         |
| Actualizar una incidencia existente               |       ✅ Sí       | @Jira actualiza PROY-123 indicando que el problema está resuelto |
| Cambiar la prioridad de una incidencia            |       ✅ Sí       | @Jira cambia la prioridad de PROY-123 a Critical                 |
| Asignar una tarea a otro usuario                  |       ✅ Sí       | @Jira asigna PROY-123 a @usuario                                 |
| Crear una incidencia a partir de un hilo de Slack |       ✅ Sí       | @Jira crea una tarea con la información de este hilo           |


Con Jira Cloud para Slack tendrás un **flujo directo desde la conversación** hasta la tarea, sin que nadie tenga que abrir Jira durante el proceso. ¡Te invitamos a probarla!

Esta <a href="https://community.atlassian.com/forums/Jira-articles/Capture-assign-and-execute-work-in-Jira-from-Slack/ba-p/3252402" target="_blank" rel="nofollow noopener noreferrer">funcionalidad</a> ha sido anunciada por Atlassian y está disponible para Jira Cloud mediante la aplicación Jira Cloud para Slack.

Si tu equipo utiliza Slack como canal principal de comunicación, la integración Jira Cloud para Slack permite convertir conversaciones en tareas sin cambiar de aplicación. Esto reduce el tiempo dedicado a tareas administrativas y facilita que incidencias, decisiones y cambios de alcance queden registrados directamente en Jira.
